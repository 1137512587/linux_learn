## IPC对象的持续性
随进程持续：
一直存在，直到最后一个进程结束。（如pipe和FIFO）
随内核持续：
一直存在，直到内核自举或显示删除。（如system v消息队列、共享内存、信号量）
随文件系统持续：
一直存在，直到显示删除，即使内核自觉还存在。（POSIX消息队列、共享内存、信号量如果是使用映射文件来实现）

# 死锁
死锁是互斥进程存在的一种问题，是指多个进程之间相互等待对方资源，而在得到对方资源之前又不释放自己占用的资源，从而造成循环等待的现象。
